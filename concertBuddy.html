Concert Buddy

Designing a smarter, more social concert-going experience

Project Overview

Concert Buddy is a mobile application that empowers concert-goers to plan events, coordinate with friends, and capture memories seamlessly. Our team of five tackled this challenge over a 9-week sprint with a focus on personalized and group-based user journeys.

Challenge

Planning concerts with friends is chaotic—from discovering shows to organizing logistics. Concert Buddy aims to centralize and simplify this experience for users.

Goals

• Streamline the concert discovery and planning process.
• Enable group coordination and shared memory capture.
• Provide contextual, real-time insights and reminders.

  • 54% prefer ride-shares over parking hassles

  • 88% focus on artists over venues

User Journey Mapping

Mapping user emotions from discovery to reflection revealed moments of delight and pain. The Concert Buddy experience is designed to elevate emotional highs and address friction points.

Design Process

  1. Discovery: user interviews, affinity mapping, and persona spectrum development
  2. Design Phase I: moodboards, information architecture, and wireframes
  3. Testing Phase I: observational testing, AEIOU framework, priority matrix
  4. Design Phase II: visual system, high-fidelity prototyping
  5. Delivery: Figma prototype, team report and documentation

Feature Prioritization

To ensure the MVP focused on real user needs, we used the MoSCoW framework to prioritize features such as:

  • Must: Personalized concert discovery, group coordination, memory timeline
  • Should: Real-time alerts, post-concert sharing features
  • Could: Venue ratings, fan meetups, public photo albums
  • Won’t: Ticketing integration (deferred for future phases)

Usability Testing Findings

  • 5 out of 6 users successfully completed all core tasks
  • Users appreciated the group planning feature and visual event cards
  • Feedback led to improvements in navigation flow and confirmation prompts
  • One key pain point was visibility of shared tasks during event planning

Final Deliverables

Based on testing feedback, we iterated and delivered a polished high-fidelity prototype. It includes intuitive navigation, visual timelines, and personalized recommendations.

Reflections

Working on Concert Buddy reinforced the value of iterative design and empathy-driven research. Mapping out detailed user journeys early on provided a strong foundation for feature alignment. Usability testing brought clarity to our priorities and encouraged deliberate design decisions.
